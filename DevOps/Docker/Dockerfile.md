``` Dockerfile
# Указываем фундамент
FROM node:18-alpine

# Создаем папку приложения внутри образа
WORKDIR /usr/src/app

# Сначала копируем только список зависимостей (для кэширования)
COPY package.json package-lock.json ./

# Устанавливаем библиотеки
RUN npm install --only=production

# Копируем остальной код проекта
COPY . .

# Указываем порт (необязательно, для ясности)
EXPOSE 3000

# Главная команда при старте контейнера
CMD ["node", "notes-app"]

```

8. **Полные (Full)** — весят **500 МБ – 1 ГБ+**.
    - _Примеры:_ `python:3.11`, `node:20`, `ubuntu:22.04`.
    - _Плюсы:_ Внутри есть всё: компиляторы, утилиты отладки, стандартные библиотеки.
    - _Минусы:_ Огромный размер, много лишнего мусора, выше риск уязвимостей.
9. **Slim-версии** — весят **100 – 200 МБ**.
    - _Примеры:_ `python:3.11-slim`, `node:20-slim`.
    - _Плюсы:_ Это «облегченная» версия полной ОС. Удалены исходники документации и редкие пакеты, но основные системные библиотеки на месте.
    - _Минусы:_ Если для работы кода нужна специфическая системная утилита (например, `curl` или `git`), её придется доустановить через `RUN apt install`.
10. **Alpine (Ultra-light)** — весят **5 – 50 МБ**.
    - _Примеры:_ `python:3.11-alpine`, `node:20-alpine`, `alpine:latest`.
    - _Плюсы:_ Невероятно маленькие. Идеальны для микросервисов и безопасности.
    - _Минусы:_ Используют библиотеку `musl` вместо стандартной `glibc`. Из-за этого некоторые Python-библиотеки или бинарные файлы могут работать медленнее или требовать сложной ручной сборки.
